HOW TO DOWNLOAD AND RUN THIS PROJECT
What the Headlines Say — CS 210 Final Project
Sadhana Vasanthakumar | Rutgers University | Spring 2026

READ THIS FIRST — THE ZIP DOWNLOAD WILL NOT WORK ON ITS OWN
--------------------------------------------------------------------------------

This project uses Git LFS (Large File Storage) to store large data files.
If you download the ZIP from GitHub or Codebench, every CSV and database file
will appear as a 3-line text placeholder like this:

    version https://git-lfs.github.com/spec/v1
    oid sha256:122c12623c6381cdf89d5cdfef5c120691676d30...
    size 85715434

That is NOT the real file. You must follow the steps below to get the actual
data. The only method that works is git clone + git lfs pull.

ALSO: Please check all Codebench submissions, not just the most recent one.
Earlier submissions may contain output files not present in the latest upload.
The most complete and up-to-date version of everything is always on GitHub.

STEP 0 — VERIFY YOU HAVE PYTHON 3.10+
================================================================================

    python --version
    # or
    python3 --version

If you are below 3.10, install a newer version from python.org before continuing.

STEP 1 — INSTALL GIT LFS
================================================================================

This is a one-time setup on your machine. Skip if you already have it.

macOS (Homebrew):
    brew install git-lfs
    git lfs install

Ubuntu / Debian / WSL:
    sudo apt install git-lfs
    git lfs install

Windows:
    Download the installer from https://git-lfs.com
    Run the installer, then open Git Bash and run:
    git lfs install

IMPORTANT: "git lfs install" must be run after installing the binary.
Without it, Git will not know to pull LFS files even if the package is
installed. This is the most common setup mistake.

To verify LFS is ready:
    git lfs version
    # Should print something like: git-lfs/3.x.x (...)

STEP 2 — CLONE THE REPOSITORY
================================================================================

Do NOT use the "Download ZIP" button. Use git clone:

    git clone https://github.com/SadhanaVasanthakumar/sv903_FinalProject.git
    cd sv903_FinalProject

After this step the notebooks and HTML file are present, but the large data
files are still pointer text. Do not open any notebooks yet.

STEP 3 — PULL THE LARGE DATA FILES
================================================================================

    git lfs pull

This downloads the actual CSVs and the SQLite database. Total size is roughly
300 MB. Expect 1–3 minutes on a normal connection.

Files downloaded by this step:
    data/ai_headlines_full.csv       (~84 MB)  ← required for TA demo
    data/ai_headlines_raw.csv        (~19 MB)
    data/bias_observatory.db         (~103 MB)
    data/figshare_headlines.csv      (~12 MB)
    data/figshare_with_features.csv  (~28 MB)
    + all output CSVs in outputs/    (~50 MB)

HOW TO VERIFY THE PULL WORKED:
Run this one-liner from the repo root:

    python -c "import pandas as pd; df = pd.read_csv('data/ai_headlines_full.csv', nrows=3); print(df)"

    GOOD — you see 3 rows with headline text, publisher names, and dates.
    BAD  — you see "version https://git-lfs.github.com/spec/v1"
           If bad: re-run "git lfs pull" and check the troubleshooting
           section at the bottom of this file.

STEP 4 — INSTALL PYTHON DEPENDENCIES
================================================================================

    pip install -r requirements.txt
    python -m nltk.downloader vader_lexicon

All packages are pinned to exact versions in requirements.txt.

NOTE: The sentence-transformers model (~80 MB) is NOT in requirements.txt.
It downloads automatically from HuggingFace the first time Notebook 05 runs.
You need an internet connection for that first run only. After that it is
cached in ~/.cache/huggingface/ and runs offline.

STEP 5 — RUN THE TA DEMO (NOTEBOOK 05)
================================================================================

This is the only notebook you need to run for grading.

    jupyter notebook 05_ta_demo.ipynb
    # Then: Kernel → Restart & Run All

Expected runtime: 8–12 minutes on CPU.
The sentence transformer encoding step (~60–90 seconds) is the bottleneck.

When it finishes, check this file for a one-page summary of all results:
    outputs/demo/demo_summary.txt

Files the demo generates (in outputs/demo/):
    features_demo.csv          NLP feature table for the 500-headline sample
    monthly_trend_demo.csv     Monthly PSI and AAI aggregates
    demo_summary.txt           Full text summary of every result

WHAT THE TA DEMO RUNS (summary)
================================================================================

Section 0   Draws 500 headlines from ai_headlines_full.csv (stratified by
            monthly window, random_state=42). Same 500 headlines every run.

Section 1   Builds data/demo_observatory.db — same 4-table SQLite schema
            as Notebook 01, populated with the 500-headline sample.

Section 2   Full NLP pipeline: VADER sentiment, PSI classification, AAI
            scoring, 8 cognitive bias dimensions, Invisible Human detection.

Section 3   Ridge regression (AAI prediction), Decision Tree (bias type
            classification), K-means clustering. Reports R², CV scores, k.

Section 4   PSI spot-check (5 headlines per class) and Invisible Human
            control corpus comparison against a 30-headline climate set.

TROUBLESHOOTING
================================================================================

SYMPTOM: CSV opens as 3 lines starting with "version https://"
CAUSE:   git lfs pull did not run or failed silently
FIX:     Run "git lfs pull" again from the repo root

SYMPTOM: git lfs pull returns HTTP 403
CAUSE:   GitHub LFS monthly bandwidth has been exhausted (1 GB free/month;
         ~300 MB per clone means only 3 full pulls before the limit hits)
FIX:     Contact sadhana — she will provide the files directly

SYMPTOM: git lfs: command not found
CAUSE:   LFS is installed but "git lfs install" was never run
FIX:     Run "git lfs install" then retry "git lfs pull"

SYMPTOM: FileNotFoundError: data/ai_headlines_full.csv
CAUSE:   Jupyter was launched from the wrong directory
FIX:     cd into the repo root folder, then run "jupyter notebook"

SYMPTOM: ModuleNotFoundError: sentence_transformers
CAUSE:   requirements.txt install incomplete
FIX:     pip install sentence-transformers

SYMPTOM: LookupError: vader_lexicon not found
CAUSE:   NLTK data not downloaded
FIX:     python -m nltk.downloader vader_lexicon
